An interactive web application inspired by the classic “Guess Who?” game, designed to explore how people express stereotypes. Instead of discussing stereotypes explicitly, players reveal their reasoning through the process of asking yes/no questions and eliminating characters — while a moderator observes and records both choices and dialogue for analysis.


---

## ✨ Features 

- 🧑‍🎓 **Player 1** – sees one *secret* card and answers yes/no questions  
- 🧑‍🚀 **Player 2** – sees all character cards, asks questions, and eliminates options based on the answers
- 🧑‍⚖️ **Moderator** – monitors both players in real time, asks questions, and can start or reset games.
- 💬 **Real-time communication** powered by Socket.IO (synchronized questions, answers, and card eliminations).
- 🗃️ **SQLite logging** of all events (chat, eliminations, system messages)

## 🚀 How to Run / Test

### 1️⃣ Set up environment
```bash
# Create and activate a virtual environment
python -m venv venv
source venv/bin/activate   # (Windows: venv\Scripts\activate)

# Install required packages
pip install -r requirements.txt
```

### 2️⃣ Start the server
```bash
python app.py
```
### 3️⃣ Create and Join a Game Session
Each game has its own unique `game_id` generated by the backend.

1. **Open the main index page**  

```
http://127.0.0.1:5000/
```
Here the **Moderator** can log in using a password and create a new game session.

2. **Click “New Game.”**  
This triggers the `/create_game` route, which:
- Creates a new record in the database  
- Generates a unique `game_id`  
- Displays **three shareable links** for:
  - Player 1  
  - Player 2  
  - Moderator view

## Data and Logging

Each game session produces structured logs for analyzing player behavior and interaction patterns.

- **Event logs** — stored in `db/games.db`, containing:
  - `game_id`, timestamps, and player identifiers  
  - Card elimination sequence  
  - Yes/No responses and moderator questions  
  - System messages and session metadata  

- **Audio recordings** *(future)* — captured via WebRTC and saved per session as `.webm` files.

- **Analysis** — logs can be exported or queried in Python, R, or SQL to examine how stereotypes emerge implicitly through gameplay.
